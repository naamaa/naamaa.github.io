Verkkoprotokollat

Alta löydät mielestäni tärkeitä verkkoprotokollia ja infoa niistä.

IP, internet protocol

  • Internetin ydin, yllättävästi siis erityisen tärkeä protokolla
  • Huolehtii IP-pakettien toimittamisesta internet-verkossa
  • Yhdistää kaikkia ja kaikkea internetissä.
  • Kuva: https://fi.wikipedia.org/wiki/IP

HTTP-protokolla

  • Hypertext transfer Protocol, hypertekstin siirtoprotokolla
  • Verkkosovelluskehittäjän kannalta ilmeisen olennainen protokolla, jota selaimet sekä WWW-palvelimet käyttävät tiedonsiirtoon
  • Client (vaikkapa selain) avaa TCP-yhteyden palvelimelle ja lähettää pyynnön, johon palvelin vastaa esimerkiksi HTML-sivulla.
  • HTML-sivua pyytäessä lähetetään palvelimelle GET-pyyntö, johon palvelin vastaa.

HTTP-protokolla, metodit

  • Tyypillisimpiä metodeita:
    • GET - resurssin haku
    • POST - lähetys (vaikka lomakkeen)
    • DELETE - poisto!

SOAP-protokolla

  • Suuren ja mahtavan Microsoftin kehittämä
  • RCP-tyyppinen, mutta pohjautuu XML kieleen ja toimii useiden eri protokollien yli
  • Käytetään pääasiassa aikaisemmin esitellyn HTTP-protokollan yli.
  • Kuva: https://fi.wikipedia.org/wiki/SOAP#/media/File:SOAP.svg

FTP-protokolla

  • File transfer protocol
  • Nimensä mukaisesti tiedostojen siirtelyyn tarkoitettu protokolla
  • Ohjelmoijan kannalta oleellinen. Lokaalisti kehitetyt tiedostot on jossain vaiheessa saatava palvelimelle.
  • Näppärä ohjelmoija muodostaa yhteyden palvelimeen, jonka jälkeen voidaan käpistellä palvelimella olevia tiedostoja tai siirtää sinne uusia.
  • Ollessaan yksinkertainen, FTP ei itsessään sisällä tiedostojen / tietojen salaamista. Tästä johtuen salaaminen on hoidettava toisella tapaa, joko kehittyneemmällä FTP-palvelimella tai tietyillä ohjelmistoilla.
